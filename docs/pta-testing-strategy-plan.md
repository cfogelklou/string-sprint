# PTA Test Mode — Simplified Plan

> **Note**: This is a historical planning artifact. The implementation described
> below has been completed. Line numbers and specific removals may not match the
> current codebase. For the current testing strategy, see `testing-strategy.md`.

## Context (Plain English)

**Fakepiano = speaker. Strobopro = listener.**

Fakepiano plays a piano tone with a **known, mathematically exact** inharmonicity (called "B"). Strobopro listens to that tone and tries to measure B. The game is: **how close did strobopro get?**

Strobopro's PTA feature has a 5-step wizard. We want to add a matching "PTA Test" mode to fakepiano so you can test each step of that wizard against known ground truth.

### What we already have (no work needed)

- ✅ **6 piano profiles** with all 88 B values pre-computed (the "answer key") — `src/bCoefficients/profiles.ts`
- ✅ **Audio engine** that plays tones using those exact B values — `src/audio/audioEngine.ts`
- ✅ **B curve editor** to visualize B values across the keyboard — `src/components/BCurveEditor.tsx`
- ✅ **Tuning sim game** with scoring — `src/hooks/useTuningSimulation.ts`
- ✅ **Stretch targets** (Railsback curve, partial alignment) — `src/tuning/stretchTargets.ts`

### What PTA tests that we don't support yet

| PTA Step | What it does | What we need to add | Why |
|----------|-------------|-------------------|-----|
| 1. Setup | Pick piano type, octave style, reference frequency | Octave style selector + reference freq control | Currently we can't change A4 from 440Hz or tell strobopro which alignment style to use |
| 2. Bridge Break | Find where thick wound bass strings transition to thin plain wire treble strings | **Informative step**: show the bridge break note for the selected piano type, explain what it is (visible on real piano — wound strings are thick copper-wrapped, plain wire is thin steel), explain why it matters (splits B interpolation into bass/treble sections). User can optionally adjust the value. No guessing game — just education. | This teaches the user what bridge break means so they can find it on a real piano |
| 3. Measure Samples | Play 8 specific notes, measure their B coefficients | Play each note → user types strobopro's measured B into a number input → we compare to the known true B from `PIANO_B_PROFILES` | This is the core test: is strobopro measuring B correctly? |
| 4. Review | Show the resulting stretch curve | Show the expected curve (generated by `calculateRailsbackOffset` using the B values from `PIANO_B_PROFILES` and the octave style from step 1) so user can visually compare with strobopro's output | Visual confirmation that everything looks right |

### Step 4: Create `src/store/ptaStore.ts`

Separate Zustand store for PTA wizard state:
- `ptaActive: boolean` — whether PTA wizard is open
- `ptaState`: wizard step, piano type, octave style, bridge break note, sample measurements, generated stretch curve, grade
- `startPTAMode()`: calls `usePianoStore.getState().stopTuningSim()` first if sim active, then initializes PTA state
- `stopPTAMode()`: sets ptaActive=false, stops all tones, clears PTA state
- `ptaSetPianoType(type)`: loads B profile into pianoStore
- `ptaSetOctaveStyle(style)`
- `ptaSetBridgeBreak(midiNote)`: updates bridge break (informational, user can adjust)
- `ptaCaptureSample(midi, measuredB)`: looks up true B from `PIANO_B_PROFILES` at index `midi - MIDI_A0`, computes error, stores measurement
- `ptaGenerateCurve()`: calls `calculateRailsbackOffset(PIANO_B_PROFILES[type], octaveStyle, referenceFreq)` for review step
- `ptaGoToStep(step)`

### Step 5: Build PTA wizard UI

- `PTAWizard.tsx` — bottom sheet overlay, swipe-to-dismiss (copy BCurveEditor pattern), step indicator (4 dots for setup/break/measure/review), z-index above BCurveEditor
- `PTASetupStep.tsx` — 6 piano type chips, 4 octave style chips, ref freq number input with presets (440/442/443)
- `PTABridgeBreakStep.tsx` — educational text explaining wound vs plain strings, shows default bridge break note for selected type, keyboard highlight, optional slider to adjust
- `PTAMeasureStep.tsx` — 8 sample notes listed, each with play button + number input for B value (validated 0.00001–0.1), shows error % and color (green/yellow/red), min 3 captured to proceed
- `PTAReviewStep.tsx` — canvas chart showing expected stretch curve (reuse BCurveEditor drawing logic), summary stats
- `PTAResultsStep.tsx` — overall grade based on measurement score, reuse existing grade color palette (#00e676/#ffaa00/#ff4444)

### Step 6: Wire it up
- Add "PTA Test" button to `TuningSimPanel.tsx` — visible when tuning sim phase is 'idle', alongside existing "Start Game" controls
- Add PTAWizard overlay to `App.tsx` — `{ptaActive && <PTAWizard />}`, same render pattern as BCurveEditor
- PTA and tuning sim are mutually exclusive: starting PTA auto-stops tuning sim
